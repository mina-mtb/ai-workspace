# Broker Configuration

این فایل ساختار مفهومی و مسیر اجرای Gateway/Broker را برای تمام فراخوانی‌های MCP تعریف می‌کند.

## مسیر اجرا (Execution Flow)
هر درخواست از سمت ایجنت یا کلاینت به سرور MCP باید مراحل زیر را طی کند:

1. **Permission Check:** آیا ایجنت درخواست‌کننده به این قابلیت/سرور دسترسی دارد؟ (با ارجاع به `permissions.md`)
2. **Input Validation:** بررسی تطابق Schema ورودی (جهت جلوگیری از تزریق یا پارامترهای مخرب).
3. **Execution:** اجرای ابزار روی سرور مربوطه.
4. **Output Validation:** اعتبارسنجی خروجی (جهت جلوگیری از نشت داده حساس).
5. **Audit Logging:** لاگ کردن عملیات در `audit-policy.md`.

*(یادداشت: این مکانیزم مفهومی است و با پیاده‌سازی‌های عملی و پراکسی‌های MCP در آینده ادغام می‌شود.)*
