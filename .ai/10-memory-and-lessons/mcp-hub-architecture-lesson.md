# Lesson: MCP Hub Architecture & Fallback Pattern

**Date:** 2026-07-01
**Context:** پیاده‌سازی بانک مرکزی MCP برای فضای کاری AI-Workspace.

## مفاهیم کلیدی و درس‌آموخته‌ها
۱. **MCP به عنوان ابزار، نه عامل مستقل:** سرورهای MCP صرفاً مجموعه‌ای از قابلیت‌ها را ارائه می‌دهند (مثل یک جعبه‌ابزار) و تصمیم‌گیری نهایی همواره بر عهده Agent (Orchestrator) و با استفاده از مکانیزم‌های کنترلی Gateway است.
۲. **الگوی Fallback برای تاب‌آوری (Resilience):** برای سرویس‌های مهم مثل Jira، نباید فقط به یک تامین‌کننده (Provider) متکی بود. استفاده از الگوی `capability` با چندین پروایدر و اجرای سوییچ کنترل‌شده (READ به صورت خودکار، WRITE با تأیید انسان) قابلیت اطمینان سیستم را تضمین می‌کند.
۳. **امنیت و جداسازی Config از Secrets:** تمام تنظیمات محیطی از کدهای منبع جدا شده و در مخزن `-config` مستند می‌شوند، در حالی که Secretها در هیچ ریپازیتوری نگهداری نخواهند شد. این الگو (Rule 15) به عنوان یک استاندارد امنیتی برای تمام پروژه‌های آتی تثبیت گردید.
۴. **خودیادگیری با شرط انسان:** قابلیت ایجنت‌ها برای ساخت ابزار جدید بسیار قدرتمند است، اما در نقاط حساس باید حتماً توسط یک انسان نظارت و تأیید شود (Human-in-the-loop).
