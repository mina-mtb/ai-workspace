# Workflow: Create New MCP Server

این گردش‌کار نحوه اضافه کردن یک سرور/ابزار جدید به سیستم مرکزی (MCP Hub) را توضیح می‌دهد.

## مراحل
1. **تشخیص نیاز:** Orchestrator متوجه می‌شود که ابزار یا سرور درخواستی در `catalog.md` وجود ندارد.
2. **اسکلت‌بندی (Scaffold):** سرور جدید با استفاده از ساختار موجود در `templates/mcp-server-python-template/` راه‌اندازی و کدنویسی می‌شود.
3. **ثبت در Gateway:**
   - فایل `manifest.md` برای سرور جدید ساخته می‌شود.
   - پیکربندی‌های Gateway و اعتبارسنجی‌های دسترسی در `permissions.md` به‌روز می‌شوند.
4. **تست و ارزیابی:** سرور در محیط محلی و به‌صورت خشک (Dry Run) تست می‌شود.
5. **افزودن به Catalog:** در صورت موفقیت‌آمیز بودن تست‌ها، سرور رسماً در `catalog.md` ثبت می‌گردد.
6. **ثبت درس‌آموخته:** در صورتی که ابزار دارای الگویی تکرارشونده و مفید باشد، ساختار آن متبلور (Crystallize) شده و در مسیر `.ai/10-memory-and-lessons/` به عنوان تجربه ثبت می‌گردد.

## 🚨 خط قرمز (تأیید انسان)
**در هرگونه فعالیت مربوط به حوزه‌های حساس زیر، اخذ صریح تأییدیه کاربر انسانی (Human-in-the-loop) قبل از اجرا و کامیت اجباری است:**
- شبکه‌سازی و اتصالات خارج از سیستم.
- دسترسی یا نوشتن در پایگاه‌داده‌های عملیاتی (Production).
- ایجاد، خواندن یا ویرایش Secretها.
- هرگونه عملیات مخرب یا بازگشت‌ناپذیر روی فایل‌ها و تسک‌ها.
